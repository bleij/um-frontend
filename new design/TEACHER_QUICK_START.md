# 🚀 Быстрый старт - Интерфейс Учителя

## Шаг 1: Подготовка тестовых данных

### 1.1 Найдите ID вашей организации

```sql
SELECT id, organization_name FROM organization_profiles LIMIT 1;
```

Скопируйте полученный `id` организации.

### 1.2 Обновите SQL скрипт

Откройте файл `/teacher_demo_data.sql` и замените все вхождения `'org-id-here'` на реальный ID организации:

```sql
-- Было:
'org-id-here'

-- Стало (например):
'01HYZR3P2N9YQ8K7M6X5W4V3T2'
```

### 1.3 Выполните SQL скрипт

Запустите весь скрипт в вашей БД Turso или выполните через Turso CLI:

```bash
turso db shell um-figma-ndrt < teacher_demo_data.sql
```

## Шаг 4: Проверка базы данных

### Проверить сохранение посещаемости

```sql
SELECT 
  s.id,
  c.full_name,
  a.date,
  a.status,
  a.notes
FROM org_attendance a
JOIN org_students s ON a.student_id = s.id
JOIN child_profiles c ON s.child_profile_id = c.id
WHERE a.date = date('now')
ORDER BY a.created_at DESC;
```

### Проверить сохранение прогресса

```sql
SELECT 
  c.full_name,
  sp.skill,
  sp.value,
  sp.notes,
  sp.date
FROM student_progress sp
JOIN org_students s ON sp.student_id = s.id
JOIN child_profiles c ON s.child_profile_id = c.id
ORDER BY sp.created_at DESC
LIMIT 10;
```

## 🐛 Возможные проблемы

### Проблема 1: "Учитель не найден"

**Причина**: Номер телефона в БД не совпадает с введенным

**Решение**:
```sql
SELECT phone FROM org_teachers WHERE id = 'teacher-demo-001';
-- Проверьте, что номер точно '79991234567' (без пробелов и знаков)
```

---

### Проблема 2: "Нет групп"

**Причина**: Группы не привязаны к учителю

**Решение**:
```sql
SELECT * FROM org_course_groups WHERE teacher_id = 'teacher-demo-001';
-- Если пусто, выполните SQL скрипт заново
```

---

### Проблема 3: "Нет учеников в группе"

**Причина**: Ученики не записаны в группу

**Решение**:
```sql
SELECT * FROM org_students WHERE group_id IN ('group-demo-001', 'group-demo-002');
-- Если пусто, выполните SQL скрипт заново
```

---

### Проблема 4: Blank screen или ошибка

**Причина**: Ошибка в коде или отсутствие зависимостей

**Решение**:
1. Откройте DevTools (F12)
2. Проверьте Console на наличие ошибок
3. Проверьте Network вкладку на failed requests
4. Убедитесь, что `sonner` установлен: `npm list sonner`
