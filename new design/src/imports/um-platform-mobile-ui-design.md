Task:
Create a mobile Ul design for the UM platform - an intelligent ecosystem for children (6-20 years), parents, organizations, and mentors.
The platform includes child assessment tests, Al-based recommendations for clubs, scheduling, attendance tracking, gamification, mentor
support, and reporting.
Style requirements:
• Modern, clean, minimalistic
• Accent color: #6C5CE7 (violet)
• White backgrounds, rounded cards
• Soft shadows, readable typography
• Mobile-first design
• Consistent Ul components across roles
• Clear visual hierarchy
Roles to design for:
1. Parent
2. Child (6-11)
3. Teen (12-17)
4. Young adult (18-20)
5. Organization / Club Admin
6. Mentor

1. Parent Mobile Screens
Home Screen:
• Header: EduPath logo, parent avatar, notifications icon
• Children cards (name, age, avatar)
• Al Recommendations: horizontal scroll cards of suggested clubs
• Upcoming classes list (date, time, club)
• Notifications section
• Bottom navigation: Children - Calendar - Clubs - Reports - Profile
Children Screen:
• List of children
• "Add Child" button
• Each card: avatar, name, age, "Open Profile"
Child Profile:
• Info: name, age, avatar
• Soft Skills overview (radar chart)
• Completed tests
• Al recommendations
• Enrolled clubs
• Button: "Take Test"
Clubs Catalog:
• Filters: age, category, schedule, location
• Club card: photo, name, age, schedule, price, button "View Details"
Club Detail Card:
• Large club image
• Description
• Age range
• Trainer/teacher
• Schedule
• Reviews
• Price
• Button: "Enroll"
• Enrollment status: Pending → Confirmed
Calendar Screen:
• Month view with colored markers for clubs
• Clickable class for details: time, club, attendance status
Testing Screen:
• List of tests
• Progress status (completed/incomplete)
• Button: "Start Test"
• In-test view: progress bar, illustrations, multiple-choice cards
Reports Screen:
• Attendance chart
• Soft Skills trend chart
• XP/level chart
• Completed tasks

7. Style & Components Guidelines
• Rounded cards, soft shadows
• Consistent color scheme (#6C5C7 accent)
• Clean, readable fonts (Inter, SF Pro)
• Mobile-first navigation
• Clear hierarchy and spacing
• Consistent iconography for actions (chat, calendar, achievements)

8. Output Requirements
• Generate all screens for each role
• Maintain single design system
• Mobile resolution (iOS & Android)
• Include interactive components where possible (buttons, tabs, scrollable lists)
