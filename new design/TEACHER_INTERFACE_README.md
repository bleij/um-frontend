# Интерфейс Учителя - UM Platform

## 📚 Обзор

Успешно реализован полноценный интерфейс для роли Учителя в платформе UM с тремя главными вкладками:

### 🏠 Вкладка 1: Моё расписание (Home)
- **Список уроков на сегодня** с временем, названием курса и аудиторией
- **Статус следующего урока** - отображается время до начала
- **Кнопка "Сканер QR"** для активации новых учеников на их первом занятии
- **Карточки уроков** с возможностью перехода к группе

### 👥 Вкладка 2: Мои группы (Groups)
- **Список всех групп** учителя с информацией о курсе
- **Счетчик учеников** (текущее количество / максимум)
- **Расписание группы** и статус активности
- **Детальная страница группы** с двумя подвкладками:
  
  #### 📋 Посещаемость
  - Список учеников с аватарами
  - Быстрая отметка посещаемости (3 состояния):
    - ✅ Присутствует (зеленый)
    - ⏰ Опоздал (желтый)
    - ❌ Отсутствует (красный)
  - Сохранение посещаемости в БД
  - Легенда для понимания статусов
  
  #### 📈 Оценка прогресса
  - Выбор ученика из списка
  - Выбор навыка с emoji-иконками:
    - 👑 Лидерство
    - 🧩 Логика
    - 🎨 Креативность
    - 💬 Коммуникация
    - 🤝 Командная работа
    - 🎯 Решение задач
  - Поле для комментария
  - Сохранение прогресса в таблицу `student_progress`

### 👤 Вкладка 3: Профиль
- **Информация об учителе**: имя, специализация, фото
- **Рейтинг учителя** на основе отзывов родителей
- **Бейдж организации** с логотипом и названием
- **Контактная информация**: телефон и email
- **Статистика**: количество групп, учеников, рейтинг
- **Кнопка выхода** из аккаунта

## 🎨 Дизайн

- **Фиолетовый акцентный цвет**: #6C5CE7
- **Градиентные фоны**: от фиолетового к белому
- **Белые карточки** с тенями и скругленными углами
- **Адаптивность**: max-width 480px для мобильных устройств
- **Плавные анимации** и переходы
- **Tab Bar навигация** внизу экрана

## 🗄️ База данных

### Использованные таблицы:
- **org_teachers** - профили учителей
- **org_course_groups** - группы курсов
- **org_students** - ученики организации
- **org_courses** - курсы организации
- **child_profiles** - профили детей
- **parent_profiles** - профили родителей
- **org_attendance** - посещаемость

### Новая таблица:
- **student_progress** - прогресс учеников по навыкам (создается автоматически)

## 🔐 Аутентификация

Учителя входят через номер телефона (как и все пользователи):

1. Система проверяет номер в таблице `org_teachers`
2. При успешной авторизации сохраняется `teacher_id` в localStorage
3. Перенаправление на `/teacher` (главную страницу расписания)

## 🧪 Тестирование

### Для тестирования интерфейса учителя:

1. **Создайте тестового учителя в БД**:
```sql
INSERT INTO org_teachers (id, organization_id, full_name, phone, email, specialization, rating, status)
VALUES (
  'teacher-test-001',
  'org-id-here',
  'Иван Иванов',
  '79991234567',
  'ivan@example.com',
  'Математика и Программирование',
  4.8,
  'active'
);
```

2. **Создайте группу для учителя**:
```sql
INSERT INTO org_course_groups (id, course_id, group_name, teacher_id, max_students, current_students)
VALUES (
  'group-test-001',
  'course-id-here',
  'Группа А - Начинающие',
  'teacher-test-001',
  15,
  8
);
```

3. **Войдите через номер телефона учителя**:
   - Откройте `/login`
   - Введите: `+7 (999) 123-45-67`
   - Введите любой SMS-код (в демо режиме)
   - Система автоматически определит учителя и перенаправит на `/teacher`

## 📝 Примечания

- Все тосты используют библиотеку `sonner` для уведомлений
- QR-сканер использует компонент `QRScanner` (уже существующий)
- Мобильная адаптация: max-width 480px
- Tab Bar фиксирован внизу экрана (z-index: 50)
- Прогресс сохраняется с инкрементом +1 к навыку

## 🐛 Известные ограничения

1. Расписание пока генерируется на основе групп (нужна отдельная таблица)
2. Рейтинг учителя статичен (нужна система отзывов)
3. QR-активация учеников пока показывает alert (нужна полная реализация)
4. Статистика в профиле показывает "-" (нужны реальные подсчеты)
